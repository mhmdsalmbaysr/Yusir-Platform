# منصة يُسر — Yusir Platform

منصة تسوّق تفاعلية تعرض متاجر اليمن على خريطة جغرافية. عند النقر على أي متجر تنفتح لوحة جانبية تستعرض منتجاته، ويمكن للعميل إضافتها إلى السلة وإتمام الطلب — وكل ذلك يتم محلياً في جانب العميل (Client-Side) دون إعادة تحميل الصفحة.

## المعمارية (Data Flow)

```
[data/stores.geojson]
       │  (fetch غير متزامن)
       ▼
[app.js] ──► تحليل JSON وتوليد نقاط الخريطة (Markers)
       │  (حدث النقر click)
       ▼
[استخراج feature.properties]
       │  (حقن DOM التفاعلي)
       ▼
[اللوحة الجانبية Sidebar] ──► تصفّح + إضافة للسلة + إتمام الطلب
```

## الميزات
- 🗺️ خريطة تفاعلية لكل المتاجر مع مؤشرات مخصّصة (مفتوح/مغلق).
- 🛒 سلة شراء محفوظة في المتصفح (localStorage) تبقى بعد تحديث الصفحة.
- 📱 إتمام الطلب عبر واتساب — طلب منفصل لكل متجر على رقمه الخاص + احتساب التوصيل.
- 🧑‍💼 لوحة تحكم (`admin.html`) لإضافة المتاجر والمنتجات بصرياً وتصدير `stores.geojson`.

## النشر عبر GitHub Pages
الملف `.github/workflows/pages.yml` ينشر الموقع تلقائياً. لتفعيله مرة واحدة:
`Settings → Pages → Build and deployment → Source = GitHub Actions`.
بعدها سيكون الرابط: `https://mhmdsalmbaysr.github.io/Yusir-Platform/`

## بنية بيانات المتجر (stores.geojson)

كل متجر هو `Feature` يحمل مستويين:
- **geometry**: إحداثيات النقطة `[lng, lat]` لتثبيت المؤشر على الخريطة.
- **properties**: الهوية الرقمية للمتجر (`store_id`, `name`, `category`, `city`, `image`, `rating`, `open`, `delivery_fee`) إضافة إلى مصفوفة `products` — كل منتج يحمل `id`, `name`, `price`, `unit`, `image`, `in_stock`.

## المراجعات المشتركة (خلفية اختيارية عبر Supabase)
تعمل المراجعات افتراضياً عبر التخزين المحلي للمتصفح. لجعلها **مشتركة بين جميع الزوار**:

1. أنشئ مشروعاً مجانياً على [Supabase](https://supabase.com).
2. من **SQL Editor** شغّل المخطط التالي:

```sql
create table if not exists public.reviews (
  id bigint generated by default as identity primary key,
  store_id text not null,
  name text not null,
  text text not null,
  stars int not null check (stars between 1 and 5),
  created_at timestamptz default now()
);

alter table public.reviews enable row level security;

-- السماح بالقراءة للجميع
create policy "read_reviews" on public.reviews
  for select using (true);

-- السماح بإضافة مراجعة للجميع
create policy "insert_reviews" on public.reviews
  for insert with check (true);
```

3. من **Project Settings → API** انسخ `Project URL` و`anon public key`.
4. ضع القيمتين في `config.js`.

> مفتاح `anon` عام بطبيعته وآمن للنشر؛ تحميه سياسات RLS أعلاه.
